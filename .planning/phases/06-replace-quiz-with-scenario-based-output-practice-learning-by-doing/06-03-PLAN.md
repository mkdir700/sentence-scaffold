**Props interface:**

interface StepPracticeProps {

**Internal PracticeCard component** (colocated, unexported — same pattern as QuizCard in StepQuiz):

State per card:
- `answer: string` — controlled textarea value
- `submitted: boolean` — toggles input view vs feedback view
- `commentary: string | null` — from API response

Props: `{ task: { cn: string; hint: string; reference: string }; index: number }`

Each PracticeCard:
- Uses `useFeedback()` mutation hook internally (this is the one exception to "step components are purely presentational" — the feedback is per-card interactive state, not data fetching for the page)
- Shows: Chinese sentence (`task.cn`), hint badge (`task.hint`), textarea for user input, submit button
- Submit button: `disabled={feedback.isPending || submitted || !answer.trim()}`
  - `feedback.isPending` prevents double-submission (Pitfall 3)
  - `submitted` prevents re-submission after feedback shown
  - `!answer.trim()` prevents empty submission
- On submit: call `feedback.mutate({ userTranslation: answer, reference: task.reference, hint: task.hint, cn: task.cn })`
- On mutation success: set `submitted = true`, set `commentary = data.commentary`
- On mutation error: show inline error message (same pattern as other error states in app)
- After submitted === true, reveal with `animate-in fade-in`:
  - User's answer (in zinc card)
  - Reference translation (in indigo card, labeled "Reference Translation")
  - AI commentary (in emerald card, labeled "Feedback" — emerald for summaries/success per design system)
- Reference translation MUST NOT be visible before `submitted === true` (locked decision)

**StepPractice component:**
- Renders step header with number 6 badge and "Output Practice" label (same animation/style as StepQuiz)
- Shows scenario description in a Card with slightly different styling (e.g., bg-amber-50 border-amber-200, using amber for tips per design system)
- Renders all PracticeCards (not gated by currentTask — per research, "Show All" mode in Analysis.tsx shows all tasks visible, each requiring individual submission)
- Each card shows task number (1/N, 2/N, etc.)

**Styling guidelines (per CLAUDE.md design system):**

- Card, CardContent for task cards (reuse existing UI primitives)
- textarea styling: same as current StepQuiz textarea
- Indigo accent for reference, emerald for feedback, amber for scenario/hints
- No more than 2 accent colors per view — use zinc for everything else
