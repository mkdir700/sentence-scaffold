**Controllers** — each controller function: parses req, validates with Zod (POST only), calls service, sends res. NO business logic. Every `res.json()` or `res.status().json()` MUST be preceded by `return` (Pitfall 1).

1. **`server/controllers/analysis.ts`**:
   - Import `Request, Response` from `express`, `z, ZodError` from `zod`, service functions from `../services/analysis.js`
   - `handleCheckSentence(req, res)`: Validate body with `z.object({ sentence: z.string().min(3, "Sentence must be at least 3 characters") }).parse(req.body)`. Call `checkSentence(sentence)`. If null, return 404 `{ error: "Not found" }`. Otherwise return result. Catch ZodError -> 400. Catch other -> 500.
   - `handleSaveSentence(req, res)`: Validate body with `z.object({ sentence: z.string().min(1), analysis: z.record(z.unknown()) }).parse(req.body)`. Call `saveSentence(sentence, analysis as AnalysisResult)`. Return `{ success: true }`. Same error handling.
   - `handleGetAnalysisById(req, res)`: Parse `req.params.id` with parseInt + isNaN check (return 400 `{ error: "id must be a number" }` on NaN — Pitfall 4). Call `getAnalysisById(id)`. If null, return 404. Otherwise return result.
   - All function signatures: `(req: Request, res: Response): void`

2. **`server/controllers/library.ts`**:
   - `handleSaveToLibrary(req, res)`: Validate body with `z.object({ sentence: z.string().min(1) }).parse(req.body)`. Call `saveToLibrary(sentence)`. Match the existing behavior: if service indicates not found, return 404. Otherwise return the service result. Catch ZodError -> 400.
   - `handleGetSaved(req, res)`: No validation needed (GET). Call `getSaved()`, return result.
   - `handleGetHistory(req, res)`: No validation needed (GET). Call `getHistory()`, return result.

3. **`server/controllers/chunks.ts`**:
   - `handleSaveChunk(req, res)`: Validate body with `z.object({ expression: z.string().min(1), meaning: z.string().min(1), examples: z.array(z.string()) }).parse(req.body)`. Call `saveChunk(expression, meaning, examples)`. Return `{ success: true }`. Catch ZodError -> 400.
   - `handleGetChunks(req, res)`: No validation. Call `getChunks()`, return result.

**Routes** — each route file creates an `express.Router()`, registers paths, delegates to controller functions. No logic.

4. **`server/routes/analysis.ts`**:
   ```
   router.post("/check-sentence", handleCheckSentence)
   router.post("/save-sentence", handleSaveSentence)
   router.get("/analysis/:id", handleGetAnalysisById)
   export default router
   ```

5. **`server/routes/library.ts`**:
   ```
   router.post("/save", handleSaveToLibrary)
   router.get("/saved", handleGetSaved)
   router.get("/history", handleGetHistory)
   export default router
   ```

6. **`server/routes/chunks.ts`**:
   ```
   router.post("/chunks", handleSaveChunk)
   router.get("/chunks", handleGetChunks)
   export default router
   ```

All relative imports use `.js` extension (Pitfall 2). No `@/` path aliases (Pitfall 3).
